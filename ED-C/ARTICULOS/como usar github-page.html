como usar GitHub

En este artículo aprenderás paso a paso cómo crear tu primer repositorio en GitHub y transformarlo en una página web estática accesible desde cualquier navegador. Esta guía está pensada para quienes quieren publicar contenido en línea de forma rápida, gratuita y profesional, aprovechando las herramientas que ofrece GitHub Pages. Desde la creación del repositorio hasta la configuración final, te mostraré cómo convertir tu proyecto en una web funcional sin necesidad de servidores ni complicaciones técnicas.

Y por último, unos tips para llamar a otras cosas en tu código... menos a tu ex, por favor. Usa rutas claras y bien estructuradas para enlazar archivos como imágenes, estilos, scripts o páginas internas. Por ejemplo:

<!-- Enlace a otra página -->
<a href="contacto.html">Ir a Contacto</a>

<!-- Enlace a hoja de estilos -->
<link rel="stylesheet" href="estilos/main.css">

<!-- Enlace a archivo JavaScript -->
<script src="scripts/app.js"></script>

<!-- Llamar una imagen -->
<img src="imagenes/logo.png" alt="Logo del sitio">

y te saldrá algo como esto y aquí es donde podrás ver la ruta de tu archivo que quieres llamar.

Por ejemplo, podrías pensar que la ruta para acceder a tu página es algo como:

educate.n/index.html

Pero ¡ojo! No es tan simple. Para que el navegador sepa exactamente dónde buscar tu archivo, debes agregar una diagonal al inicio, así:

/educate.n/index.html

Sí, esa / al principio es clave. Es como decirle al navegador: “Ey, empieza desde la raíz del sitio y busca el repositorio llamado educate.n”. Sin esa diagonal, el navegador se pierde como si estuviera buscando en una calle equivocada.
